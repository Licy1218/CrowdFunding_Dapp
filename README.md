# 基于区块链的众筹系统

本项目在原 petshop Hardhat 示例基础上改造为众筹系统，使用 Solidity 合约保存项目、捐赠和结算信息，前端通过 Ethers.js 调用本地区块链上的合约。

原始 petshop 代码已备份到：

```shell
backups/petshop-original-20260608
```

## 已实现功能

- 创建众筹项目：项目名称、描述、分类、目标金额、截止日期，链上分配唯一 ID。
- 资金捐赠：用户可选择进行中的项目并捐赠 ETH。
- 捐赠记录：合约记录每个捐赠者地址、捐赠金额和项目已筹总额。
- 项目展示：前端展示所有项目的名称、描述、目标金额、已筹金额、剩余时间、进度状态和捐赠者列表。
- 到期结束：达到截止时间后，任何人都可以调用结束项目。
- 成功提款：项目达标后，发起人可以提取募集资金。
- 失败退款：项目未达标后，捐赠者可以取回自己的捐赠资金。
- 早鸟勋章：每个项目前 10 位不同捐赠者获得链上勋章积分，发起项目时会展示发起人的累计勋章。
- 里程碑释放：项目已筹金额达到目标后，发起人可在项目结束前提前释放 50% 目标金额；项目成功结束后再提取剩余资金。
- 项目分类筛选：支持按公益、校园、医疗、创意、其他分类筛选项目。
- 项目公告：项目发起人可以发布链上项目进展公告。
- 我的参与记录：前端展示当前钱包发起、参与和可退款的项目数量；点击统计块会筛选项目列表，最近 3 条记录可跳转到单个项目。
- 项目进度状态：展示未达标进行中、已达标可释放、已释放里程碑、待结束、成功待提款、成功已提款、失败可退款等细分状态。

## 额外功能规则说明

早鸟勋章不参与资金计算，只作为用户声誉展示：

- 第 1 至第 3 位不同捐赠者：获得 100 勋章积分。
- 第 4 至第 10 位不同捐赠者：获得 50 勋章积分。
- 用户累计勋章会显示在他发起的项目中，帮助其他捐赠者识别活跃支持者。

里程碑释放采用不破坏退款的方案：

- 未达到目标金额前，项目资金不能提前释放。
- 达到或超过目标金额后，发起人可以提前释放 50% 目标金额。
- 项目截止并成功结束后，发起人提取剩余资金。
- 如果项目最终未达到目标，资金不会被提前释放，捐赠者仍可全额退款。

## 运行方式

安装依赖：

```shell
npm install
```

运行测试：

```shell
npm test
```

启动本地区块链：

```shell
npm run node
```

在另一个终端部署合约：

```shell
npm run deploy:local
```

启动前端：

```shell
npm run dev
```
查看所有用户余额：

```shell
node -e "const { ethers } = require('ethers'); const provider = new ethers.JsonRpcProvider('http://127.0.0.1:8545'); (async () => { const accounts = await provider.send('eth_accounts', []); for (const [i, account] of accounts.entries()) { const balance = await provider.getBalance(account); console.log(`Account #${i}: ${account}  ${ethers.formatEther(balance)} ETH`); } })();"
```

打开 lite-server 输出的地址，通常是 `http://localhost:3000`。如果使用 MetaMask，请连接到本地 Hardhat 网络：RPC `http://127.0.0.1:8545`，Chain ID `31337`。
